## 对照 PRD 全文检查结果（按你提供的 4–6 节）

- **4.1 初见与抉择**：F.1.1 全屏欢迎（模糊底、Logo、欢迎语、按钮「开始探索」「这是什么？」）、F.1.2 四类角色卡片与默认业务执行均已实现；与 PRD 一致。
- **4.2 分角色高光引导**：Spotlight/Blocking/跳过二次确认/下一步+进度、**CXO 3 步**：① 如何提问（自动填入「上周销售额是多少？」）② 联网搜索 ③ **点击发送**（收尾聚光灯打在发送按钮，文案含「在界面内可追问」）；**追问引导仅在进入数据分析界面后出现**（提示 + 打字展示「为什么下降了？」再发送 + 发送按钮聚光灯）。业务负责人+看板+按需数字员工、一线 3 步、数据开发 4 步占位；最后一步主按钮「完成引导」。已按 PRD 对齐。
- **4.3 形态转换**：蒙版收缩至右下角、气泡「我会一直在这里！…」5 秒后消失已实现；收缩后蒙版为透明淡出，避免黑块遮挡。
- **4.4 亿问小助手**：帮助文档/截图上报/转人工/重新引导、文档内关键词检索（RAG 模拟）、截图模拟诊断、页面感知（当前页展示）、帮助中心兜底已实现。
- **5 埋点**：引导开始/完成/跳过等已通过 `yiwen_tour_track` 上报。
- **6 异常处理**：小屏引导卡宽度、网络兜底链接、跳过二次确认、版本更新清完成标记已实现。

- **欢迎页**：以 PRD 文案设计段为准，当前实现视为符合；若产品最终主按钮定稿为「开启旅程」，仅改按钮文案即可。

- **F.3.1**：已在 `FloatingGuideAssistant` 增加 `dimmed` 状态，由 `OnboardingTour` 的 `onComplete` 触发 5 秒半透明驻留。
- **数据开发**：已改为 Lazy；主引导 1 步（审计日志），数据源/建模/指标首次点击卡片触发引导气泡。
