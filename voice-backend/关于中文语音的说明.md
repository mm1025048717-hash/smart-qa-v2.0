# 关于中文语音的说明

## 当前情况

### Deepgram TTS 对中文的支持

Deepgram TTS 目前**主要支持英文语音模型**。虽然它可以处理中文文本，但：
- ⚠️ 默认语音模型是英文的（aura-2-helena-en 等）
- ⚠️ 中文发音可能不够自然，会有外国人说中文的感觉
- ⚠️ 这是 Deepgram TTS 的当前限制

### 其他选项

如果需要**更自然的中文发音**，建议考虑：

1. **Cartesia TTS**（之前使用过）
   - 支持中文，音质较好
   - 免费额度：每月 $10
   - 需要单独配置 API Key

2. **OpenAI TTS**
   - 支持中文，音质很好
   - 需要 OpenAI API Key
   - 按使用量付费

3. **ElevenLabs TTS**
   - 支持中文，音质最好
   - 需要 API Key
   - 价格较高

## 当前配置

- **TTS 服务**：Deepgram TTS
- **语音模型**：aura-2-helena-en（英文模型，但可以处理中文文本）
- **STT**：已配置为中文识别（zh-CN）
- **LLM**：已配置为中文回复

## 建议

如果中文发音不够自然，可以考虑：

1. **切换回 Cartesia TTS**：
   ```ini
   TTS_SERVICE=cartesia
   CARTESIA_API_KEY=your_key
   ```

2. **或者使用 OpenAI TTS**（如果已有 API Key）：
   ```ini
   TTS_SERVICE=openai
   OPENAI_API_KEY=your_key
   ```

## 关于其他问题

### 对话不流畅、不能即时处理

这可能是：
- 网络延迟
- 系统资源不足
- VAD/Turn Analyzer 配置问题

### 回复不完整

这可能是：
- LLM 响应被中断
- 网络问题导致响应截断
- TTS 处理问题

建议检查后端日志，查看具体错误信息。
