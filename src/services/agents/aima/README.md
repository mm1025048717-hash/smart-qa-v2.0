# 爱玛电动车数据部门数字员工

## 📋 概述

这是专门为爱玛电动车数据部门客户演示设计的数字员工系统。使用 LangChain 与大模型实现人性化回复，能够替代爱玛数据部门的各项工作。

## 🎯 数字员工列表

### 1. 艾玛数据专家（aima-data-master）
- **定位**：数据部门核心专家，全方位数据分析
- **技能**：销售分析、门店运营、库存管理、市场分析、客户满意度分析
- **性格**：专业、友好、细致
- **适用场景**：综合性数据分析需求

### 2. 艾玛销售分析师（aima-sales-analyst）
- **定位**：专注销售数据分析
- **技能**：销售趋势分析、区域对比、车型销售分析、销售预测
- **性格**：专业、敏锐、数据驱动
- **适用场景**：销售相关数据分析

### 3. 艾玛门店分析师（aima-store-analyst）
- **定位**：门店运营分析专家
- **技能**：门店坪效分析、库存周转、客流分析、门店排名
- **性格**：务实、细致、运营导向
- **适用场景**：门店运营数据分析

### 4. 艾玛市场分析师（aima-market-analyst）
- **定位**：市场与竞品分析专家
- **技能**：市场份额分析、竞品对比、市场趋势预测、区域市场分析
- **性格**：敏锐、前瞻、战略思维
- **适用场景**：市场分析和竞品对比

## 🏗️ 架构设计

### 目录结构
```
src/services/agents/
├── index.ts              # 统一入口，导出所有数字员工
├── agents.ts             # 通用数字员工
└── aima/                 # 爱玛电动车专用目录
    ├── aimaAgents.ts     # 爱玛数字员工配置和系统提示词
    └── README.md         # 本文件
```

### 系统提示词设计
- 使用 LangChain 风格设计
- 包含角色定位、专业技能、性格特点、业务背景
- 内置人性化回复模板和示例
- 支持专业且友好的对话风格

### 集成方式
1. **自动加载**：在 `App.tsx` 启动时自动预加载爱玛系统提示词
2. **智能匹配**：支持通过关键词自动匹配到合适的数字员工
3. **统一接口**：与通用数字员工使用相同的接口和调用方式

## 🚀 使用方法

### 1. 切换数字员工
用户可以通过以下方式使用爱玛数字员工：
- 在界面上选择"艾玛数据专家"等员工
- 通过自然语言："让我找爱玛数据分析专家"
- 通过关键词："爱玛销售分析"会自动匹配到销售分析师

### 2. 提问示例
- "分析本月电动自行车的销售情况"
- "查看各区域门店销售额排名TOP10"
- "对比爱玛与雅迪的市场表现"
- "分析门店坪效和库存周转率"

## 🔧 技术实现

### 系统提示词注入
- 在 `deepseekApi.ts` 中检测 `aima-` 前缀的 agentId
- 自动调用 `getAimaSystemPrompt()` 获取自定义提示词
- 在应用启动时预加载，确保第一次调用就能使用完整提示词

### 关键词匹配
- 在 `agents/index.ts` 中集成了爱玛的关键词映射
- 支持通过业务关键词自动匹配到合适的数字员工

### 人性化回复
- 基于 LangChain 和大模型生成自然语言回复
- 结合爱玛业务背景和专业术语
- 提供数据可视化和业务洞察

## 📝 配置说明

### customConfig 字段
每个爱玛数字员工都包含 `customConfig` 字段，用于扩展配置：
- `industry`: 行业（电动车制造）
- `company`: 公司（爱玛电动车）
- `businessDomain`: 业务领域
- `keyMetrics`: 关键指标
- `competitors`: 竞争对手列表
- `productLines`: 产品线
- `personality`: 性格和沟通风格
