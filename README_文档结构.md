## 📋 文档分类

### 部署相关
- 所有部署文档以 `部署` 或 `DEPLOYMENT` 开头
- 入口文档：`部署文档索引.md`

### 技术文档
- 所有技术文档位于 `docs/` 目录
- 包含 PRD、设计文档、使用指南等

### 开发文档
- `README.md` - 项目概述
- `CONTRIBUTING.md` - 贡献指南

## 🔄 文档更新原则

1. **部署文档**：统一在根目录，通过 `部署文档索引.md` 导航
2. **技术文档**：统一在 `docs/` 目录
3. **临时文档**：及时删除，避免冗余
4. **重复文档**：合并或删除，保持单一来源

## 📝 注意事项

- ⚠️ 不要创建重复的部署文档
- ⚠️ 临时整理文档应及时删除
- ✅ 使用 `部署文档索引.md` 作为部署文档的统一入口
- ✅ 技术文档统一放在 `docs/` 目录
