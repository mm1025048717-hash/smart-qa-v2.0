# 贡献指南

感谢你对项目的兴趣！我们欢迎各种形式的贡献。

## 如何贡献

### 报告 Bug

如果你发现了 bug，请：
1. 检查 [Issues](https://github.com/your-username/your-repo/issues) 中是否已有相关问题
2. 如果没有，请创建新的 Issue，使用 Bug Report 模板
3. 提供详细的复现步骤和环境信息

### 提出新功能

如果你有功能建议，请：
1. 检查 [Issues](https://github.com/your-username/your-repo/issues) 中是否已有相关建议
2. 如果没有，请创建新的 Issue，使用 Feature Request 模板
3. 详细描述功能的使用场景和价值

### 提交代码

1. Fork 本仓库
2. 创建功能分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 创建 Pull Request

## 代码规范

- 使用 TypeScript 编写代码
- 遵循 ESLint 规则
- 添加必要的注释
- 保持代码简洁和可读性

## 开发环境设置

1. Fork 并克隆仓库
2. 安装依赖：`npm install`
3. 创建 `.env` 文件并配置 API Key
4. 启动开发服务器：`npm run dev`

## 问题反馈

如有任何问题，请通过 Issues 反馈。
